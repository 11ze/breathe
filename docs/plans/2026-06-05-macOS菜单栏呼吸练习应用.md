# macOS 菜单栏呼吸练习应用（Breathe）

## Context

[breathe-cli](https://github.com/marekkowalczyk/breathe-cli) 是一个基于终端的共振呼吸引导工具（Python 单文件，零依赖），科学依据充分（6 次/分钟，vagal tone 训练）。用户希望将其完整移植为原生 macOS 菜单栏应用，复用 [menu-bar-executor](../menu-bar-executor/) 的架构模式。

**目标**：一个常驻菜单栏的 macOS 原生应用，点击图标弹出呼吸引导面板，包含动画呼吸圆环、三种预设（按时段自动选择）、自定义比例、音频提示、会话记录、定时提醒。

## 核心文件设计

### 1. `BreatheApp.swift` — 入口

```swift
@main
struct BreatheApp: App {
    @NSApplicationDelegateAdaptor(AppDelegate.self) var appDelegate
    var body: some Scene {
        Settings { EmptyView() }
    }
}
```

与 menu-bar-executor 的 `MenuBarExecutorApp.swift` 一致，最小化入口。

### 2. `AppDelegate.swift` — 菜单栏

复用 menu-bar-executor 的 `AppDelegate.swift` 模式：
- `NSStatusItem` + template 图标（自动适配明暗模式）
- 左键：toggle 呼吸面板
- 右键：NSMenu 菜单

菜单结构：
```
├── 开始呼吸              ⌘B
├── ─────────
├── 预设 ▸  自动 (按时段)
│            balanced (5-5, 10分钟)
│            calm (4-6, 15分钟)
│            extended (4-6, 20分钟)
│            ─────────
│            自定义...
├── ─────────
├── 静音                  ✓
├── 历史记录              ⌘H
├── 设置...               ⌘,
├── ─────────
└── 退出                  ⌘Q
```

"开始呼吸"在会话中变为"停止呼吸"。通过 `NSMenuDelegate` 的 `menuNeedsUpdate(_:)` 动态更新。

### 3. `BreathingEngine.swift` — 核心状态机

移植 breathe-cli 的 `run_session()` 循环，用 `Timer.scheduledTimer` (20Hz) 替代 Python 的 `time.sleep`：

```swift
enum BreathingPhase: Equatable {
    case idle
    case countdown(Int)        // 3-2-1
    case inhale
    case exhale
    case paused(BreathingPhase) // 记住暂停时的阶段
}

@MainActor
final class BreathingEngine: ObservableObject {
    static let shared = BreathingEngine()

    @Published private(set) var phase: BreathingPhase = .idle
    @Published private(set) var phaseProgress: Double = 0.0  // 0.0~1.0
    @Published private(set) var breathsCompleted: Int = 0
    @Published private(set) var remainingSeconds: Int = 0
    @Published private(set) var totalDurationSeconds: Int = 0
    @Published private(set) var isSessionActive: Bool = false

    // 20Hz tick: 计算 phaseElapsed, 切换阶段, 更新 remainingSeconds
    // 关键算术移植自 breathe-cli:
    //   breathingBase += cycleSeconds (每次完整呼吸后)
    //   remainingSeconds = duration - breathingBase - phaseElapsed (考虑当前阶段)
    //   暂停恢复：回到 INHALE 起始点 (与 CLI 行为一致)
}
```

**关键移植点**：
- 时长向上取整到完整呼吸周期：`duration = -(-totalSeconds / cycle) * cycle`
- `breathingBase` 始终是 `cycleSeconds` 的整数倍
- 剩余时间单调递减，精确到周期末尾归零

### 4. `RatioValidator.swift` — 安全约束

完整移植 breathe-cli 的 `parse_ratio()` 安全规则：

| 规则 | 约束 |
|------|------|
| 禁止屏息 | 拒绝三段比例如 `4-7-8` |
| 禁止快速呼吸 | 总周期 ≥ 8 秒 |
| 吸气范围 | 3~10 秒 |
| 呼气范围 | 3~10 秒 |
| 呼气上限 | 呼气 ≤ 2×吸气 |

### 5. `Preset.swift` — 预设定义

```swift
enum Preset: String, CaseIterable, Codable {
    case balanced    // 5-5, 10min, 6bpm (早晨)
    case calm        // 4-6, 15min, 6bpm (傍晚)
    case extended    // 4-6, 20min, 6bpm (下午)
}
```

不变量（测试覆盖）：全部 6.0 bpm，周期 ≥ 8s，时长可被周期整除。

### 6. `BreathingCircle.swift` — 动画圆环

这是 GUI 版本的标志性视觉元素，替代 CLI 的文字进度条：

- 吸气：圆环从 30% 缩放至 100%（青色渐变）
- 呼气：圆环从 100% 缩至 30%（绿色渐变）
- 暂停：停在 50%
- 空闲：30% 灰色

用 SwiftUI 的 `scaleEffect` + `.animation(.linear(duration: 0.05))` 实现流畅的 20Hz 更新。

### 7. `BreathingPanelView.swift` — 呼吸面板

布局（自上而下）：
1. **头部**：`calm · 4-6 · 14:32`（匹配 CLI 格式）
2. **呼吸圆环**：200×200pt 动画圆
3. **阶段标签**：「吸气」/「呼气」+ 倒计时秒数（大字体）
4. **进度条**：整体会话进度
5. **控制按钮**：暂停/继续、静音、停止

空闲时显示「开始呼吸」按钮 + 当前预设信息。

### 8. `BreathingPanelWindowController.swift` — 面板窗口

复用 menu-bar-executor 的 `CommandPaletteWindowController` 模式：
- `KeyPanel: NSPanel`（无标题栏，可成为 key window）
- `.floating` 级别，锚定在状态栏图标下方
- **关键区别**：会话期间**不自动关闭**（不像 menu-bar-executor 失焦即关闭），因为用户需要一边做其他事一边看呼吸引导
- 空闲状态下点击面板外部可关闭

### 9. `AppSettings.swift` — 配置管理

```swift
struct AppSettings: Codable {
    var defaultPreset: String = "auto"        // auto/balanced/calm/extended/custom
    var customInhaleSeconds: Int = 5
    var customExhaleSeconds: Int = 5
    var customDurationMinutes: Int = 10
    var soundEnabled: Bool = true
    var logSessions: Bool = true
    var launchAtLogin: Bool = false
    var dailyReminderEnabled: Bool = false
    var dailyReminderTime: String = "08:00"
}
```

Manager 复用 menu-bar-executor 的模式：原子写入 + 文件监听 + `isLoaded` 保护。

### 10. `SessionLogger.swift` — CSV 会话记录

CSV 格式与 breathe-cli 完全兼容：
```
date,time,preset,ratio,duration_target_s,duration_actual_s,breaths,completion_pct,status
2026-06-05,14:32:00,calm,4-6,900,900,90,100,completed
```

支持从旧版 `~/.breathe_log.csv` 导入历史记录。

### 11. `AudioManager.swift` — 音频

用 `NSSound` 播放系统音效，匹配 breathe-cli 的音频方案：
- 吸气：`Tink`（对应 CLI 的 `/System/Library/Sounds/Tink.aiff`）
- 呼气：`Pop`（对应 CLI 的 `/System/Library/Sounds/Pop.aiff`）
- 音量：30%

## 验证方案

1. **单元测试**：`xcodebuild test` — 覆盖安全验证、预设不变量、引擎算术、配置持久化
2. **手动集成测试**：
   - 菜单栏图标显示，明暗模式自适应
   - 左键弹出/关闭呼吸面板
   - 3-2-1 倒计时 → 吸气（圆环放大，青色）→ 呼气（圆环缩小，绿色）循环
   - 暂停：圆环停在中间，继续后从吸气重新开始
   - 静音：无音频提示
   - 会话完成后弹出通知，记录写入 CSV
   - 设置面板：切换预设、自定义比例（含安全校验反馈）、开关音频/日志/自启
   - 历史面板：显示过往会话，支持导入旧版 `~/.breathe_log.csv`
3. **CSV 兼容性**：对比导出格式与 breathe-cli 原版的 `~/.breathe_log.csv` 列定义
